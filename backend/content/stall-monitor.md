Unlike the ETA Monitor which watches predicted completion time, the Stall Monitor measures **actual progress**. If a torrent hasn't downloaded a minimum amount of data within a rolling time window, it's considered stalled and removed. This catches torrents that appear active — showing a non-zero speed or a healthy ETA — but are in practice making negligible real progress.

The check window and minimum progress threshold work together: the default requires at least 100 KB of new data within a 1-hour window. If any progress is made the baseline resets, so a torrent making slow but steady progress won't be flagged. Only genuinely stuck downloads — ones that have barely moved within the full window — are removed. If you find legitimate slow downloads being caught, either increase the window duration or lower the minimum progress threshold.
